Postgres pg_ctl vs psql

2022-06-20
 postgresdatabase

This article is based on my experience using these commands and my general understanding, there undoubtedly is more that can be accomplished with each, so please keep that in mind as you proceed through this wildly fun postgres post.

pg_ctl

Any time I reach for the pg_ctl command it is for administrating my postgres server. An example would be starting and stopping the server. The most common commands I‚Äôve used:

Start postgres server:

pg_ctl start

Stop postgres server:

pg_ctl stop

Check the status of the postgres server:

pg_ctl status

Restart the postgres server:

pg_ctl restart

psql

The psql command is used for accessing a database server. The documentation describes it as ‚Äúterminal based front-end to PostgreSQL‚Äù. There are different options you can use with this command to specify things like user and hostname. Below we cover some of the different options:

Connect to a database under root access:

psql database_name

You can also specify a specific user to connect to the database with:

psql -d database_name -U user_name -W

Or connect to a server on a different host:

psql -h host -d database_name -U user_name -W
